# Twitchy Examples

This directory contains examples showing how to use the twitchy library for building Twitch chat bots.

## Getting Your OAuth Tokens

Before running any examples, you need to obtain OAuth tokens from Twitch. There are two approaches:

### Option 1: PKCE Flow (Recommended - No Client Secret Required)

This is the easiest and most secure method for individual use. You can use a web-based token generator that implements the PKCE OAuth flow:

1. **Generate tokens** using a PKCE token generator (like the one you mentioned)
2. **Select the required scopes** based on what features you need (see below)
3. **Provide your Client ID** and redirect URI
4. **Copy the tokens** generated by the service

**Advantages:**
- No need to expose your client secret
- Perfect for sharing your app with others
- Tokens can be managed through a web service
- More secure for client-side applications

**Required scopes for basic chat:**
- `user:read:chat` - Read chat messages
- `user:write:chat` - Send chat messages (if you want to respond)

**Additional scopes for moderation:**
- `moderator:manage:chat_messages` - Delete messages
- `moderator:manage:banned_users` - Ban/timeout/unban users
- `moderator:read:chat_settings` - Read chat settings
- `moderator:manage:chat_settings` - Update chat settings

### Option 2: Traditional OAuth Flow (With Client Secret)

If you have a Twitch application with a client secret:

1. Register your application at https://dev.twitch.tv/console
2. Note your Client ID and Client Secret
3. Use the authorization code flow to obtain tokens
4. The library will automatically refresh tokens when they expire

## Examples

### [basic_chat.rs](basic_chat.rs)

The simplest example showing how to connect to Twitch chat and log messages.

**Uses PKCE flow** (client ID only, no secret required)

```bash
# Update the tokens and client_id in the file first, then:
cargo run --example basic_chat
```

**Features demonstrated:**
- Connecting to a channel
- Receiving chat messages
- Handling token expiration
- Listening to moderation events (bans, timeouts, message deletes)

### [external_token_management.rs](external_token_management.rs)

Shows how to handle tokens managed by an external service (like a web-based token generator).

**Uses PKCE flow** with manual token refresh handling

```bash
cargo run --example external_token_management
```

**Features demonstrated:**
- Using `client_id_only()` configuration
- Handling `TokenExpired` events
- Manually updating tokens with `client.update_tokens()`
- Integration with external token services

### [moderation_bot.rs](moderation_bot.rs)

A more advanced example showing moderation features.

**Can use either flow** depending on your needs

```bash
cargo run --example moderation_bot
```

**Features demonstrated:**
- Sending chat messages
- Deleting messages
- Banning/timing out users
- Advanced event handling

## Configuration

All examples follow this pattern:

```rust
use twitchy::{TwitchClient, TwitchConfig, TwitchClientEvent};

// For PKCE flow (no client secret)
let config = TwitchConfig::builder()
    .channel("channel_name")
    .tokens("your_access_token", "your_refresh_token")
    .client_id_only("your_client_id")  // No secret needed!
    .build()?;

// For automatic refresh (with client secret)
let config = TwitchConfig::builder()
    .channel("channel_name")
    .tokens("your_access_token", "your_refresh_token")
    .credentials("your_client_id", "your_client_secret")
    .build()?;
```

## Token Expiration

### PKCE Flow
When using `.client_id_only()`, you'll receive a `TwitchClientEvent::TokenExpired` event when tokens need to be refreshed. You must refresh them manually or through your token service:

```rust
TwitchClientEvent::TokenExpired => {
    // Refresh through your service, then:
    client.update_tokens("new_access_token", "new_refresh_token").await;
}
```

### Automatic Refresh
When using `.credentials()` with a client secret, the library automatically refreshes tokens. Save the new tokens when you receive them:

```rust
TwitchClientEvent::TokensRefreshed(access, refresh) => {
    // Save these tokens for next time!
    save_tokens_to_file(&access, &refresh)?;
}
```

## Troubleshooting

**"Missing scopes" warnings?**
- Make sure you selected all the required scopes when generating your tokens
- For basic chat: `user:read:chat`, `user:write:chat`
- For moderation: add the moderator scopes listed above

**"401 Unauthorized" errors?**
- Your access token may have expired
- If using PKCE flow, refresh your tokens through your token service
- If using automatic refresh, check that your client secret is correct

**"Token expired" events not being handled?**
- Make sure you're listening for `TwitchClientEvent::TokenExpired`
- Implement token refresh logic through your external service
- Call `client.update_tokens()` with the new tokens
